Hello world!

Or: How I finally decided to create my website

This website will have mostly philosophy-, psychology-, and technology-related content. I aim to display this content on a simple yet beautiful website that is designed to last and to be accessible to as many types of people as possible throughout its lifetime.

Published on September 1, 2020 • Last updated on September 5, 2020 (see update history)

Welcome! Welcome to my humble digital abode, to my corner of the internet, to another website with a forgettable name that you will bookmark but will seldomly return to.

I have several essays lined up and ready to go, but this one seems the most difficult. Maybe because it is the first one, maybe because this essay is meant to explicate the implicit assumptions of my website... maybe both. So, to achieve the goal of this essay, I will talk about:

  • what I write about,
  • how the website is built, and
  • why the website is built.

Why am I writing about this? It feels like a cliché by now, but, as Alex Nixon said, it is to justify to myself why I made this website and how.

Content

I will write about philosophy, psychology, and technology. Take a look around my website; look at my ideas, for example, and you will get a taste of what is to come, content-wise, from me.

I will write about issues I searched for but could not find a satisfactory answer to. I will also write about myself — my ideas, philosophy, stuff like that. I will try, as much as possible, to only write stuff that never existed before.

Styling

Because the medium is the message, I also have to talk about this website.

The styling was VERY HARD. I love art but I suffer from choice anxiety. There are so many different CSS frameworks to choose from and so many website styles that I like (more on this below). I settled on chota for now because of its lightness, because it provided easy instructions to write my only JavaScript file (dark-mode.js), and because it has cards.

This site is for reading, so I did not add a lot of padding — no wasted space in my layout! No animated content. No fixed floating header.

I created the favicon using Photopea and converted it to multiple formats using realfavicongenerator.net. (You can grab the .PSD file.)

The logo (my name + the favicon) was created by my friend Alex Arghiriade, because I am definetely not a web designer. I did a previous iteration myself but it did not click as much as the current logo does. The font is Segoe UI SemiLight.

I found the artwork at the top of this essay on Hacker News. There are more AI artists out there: art42.net and thisartworkdoesnotexist.com, for example.

Philosophy

You might have noticed that I like to archive every link. This is because I believe in a sustainable, long-lasting, free, and fun internet, beliefs which, I think, are shared by people like the guys at LOW←TECH MAGAZINE, Parimal Satyal, Jeff Huang, and Ben.

I also believe in an accessible internet, hence why I strived to make the website conform to some accessibility standards (like, for example, SourceHut), and why I will probably translate the content into several languages later.

I also like simplicity, balance, and durability, hence why I am writing all the pages in plain HTML with CSS I (kind of) understand and with minimal tooling overhead.

The tools in question are:

  • the VSCode IDE with the Prettier, ESLint, and Live Server extensions for writing the content;
  • GitHub and Soverin for hosting said content;
  • Microsoft's Accessibility Insights and IBM's Equal Access for accessibility testing;
  • Checkbot for assessing the SEO, speed, and security of my website; and, finally
  • Mozilla's Observatory and Google Chrome's Lighthouse for a holistic assessment.

(I found some of these tools on Shawn Wang's website which, in turn, I stumbled upon in the comments of an HN article. And I say that this is a minimal tooling overhead because I will use all tools only at the beggining, to improve my website and learn new stuff. The only long-term tools will be the first two bullet points, but even with those included I will not be affected if any/all of these tools will disappear or, worse, be discontinued. I will just change to an alternative.)

Not to be clickbaity and get to the subtitle of this essay last, but... I finally decided to make this website because I no longer feel that my ideas, poems, and essays are trash. Secondly, publishing these things pushes me to get rid of the trash. Lastly, I want an email address with a custom domain. That is it.
